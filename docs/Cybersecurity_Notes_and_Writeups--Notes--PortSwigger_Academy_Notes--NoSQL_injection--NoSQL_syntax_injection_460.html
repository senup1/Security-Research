NoSQL syntax injection


Syntax injection can be detected by attempting to break the query syntax.

To do this, assuming there are sanitization/filtering issues, test each input by submitting fuzz strings and special characters that trigger a database error or some other detectable behavior.

For MongoDB:

An example of a fuzz string (contains \ns):

'"`{

;$Foo}

$Foo \xYZ

or '%22%60%7b%0d%0a%3b%24Foo%7d%0d%0a%24Foo%20%5cxYZ%00 for web encoded or '\"`{\r;$Foo}\n$Foo \\xYZ\u0000 for JSON

To detect which characters are processed, inject characters one at a time.

After detecting one, double check by trying to escape it and seeing if the error goes away.

After detecting a vulnerability, the next step is to determine if boolean conditions have influence.

To test this, send one statement equivalent to false and one equivalent to true

For example: ‘&& 0 && ’x <or> ‘ && 1 && ’x

The next step is to inject a condition that always equates to true such as ‘||’1'=='1

This wil cause the query to return all items.

Adding a null character like %00 is treated as a kind of comment to ignore everything after it, so this can be used to prevent any additional queries from functioning.
