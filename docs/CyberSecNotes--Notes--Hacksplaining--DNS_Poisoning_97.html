DNS Poisoning


DNS poisoning occurs when a DNS server is attacked and makes valid requests to websites resolve to fake websites

A simpler attack would be modifying the victims /etc/hosts file (exists on windows and linux) to resolve to whatever ip they desire.

Slightly more complicated would be via MITM

More serious attacks target ISPs DNS resolvers.

(More details):

  • ISPs have multiple DNS Resolver servers so not everyone will be using the same one
  • Finding DNS Resolver servers is extremely easy and public (Example: just use: cat /etc/resolv.conf or ipconfig /all)

Protection:

Enabling HTTPS as this requires presenting a certificate to the victim's browser

Due to the nature of HTTPS being much more commonly used, DNS poisoning attacks are rarely used in isolation. They are usually combined with a compromised certificate
