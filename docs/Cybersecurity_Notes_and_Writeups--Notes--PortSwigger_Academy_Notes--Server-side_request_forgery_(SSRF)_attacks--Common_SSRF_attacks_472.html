Common SSRF attacks


Typically SSRF attacks involve supplying an HTTP request with a URL that points to the loopback IP which is usually 127.0.0.1 or localhost.

Example HTTP request vuln:

<usual headers>

checkAPI=<link to actual endpoint>

Chaning the link to the endpoint to localhost/admin or 127.0.0.1/admin or another directory/endpoint/target besides admin would result in the leak of sensitive information. This would only function if the owner of the request was treated as the server rather than the user assuming the server has elavated privelages.

In other cases, if the application server interacts with other back-end systems, they may use non-routable private IP addresses. In this case the attacker may attempt to scan the internal network for possible IPs to gain further access/information.
