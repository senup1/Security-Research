Using delimiter discrepancies


It is possible for different frameworks to use different delimiters.

To test for delimiter vulnerabilities, first identify what delimeter is being used.

Add an arbitrary string to the end of the URL without adding a /. If the response is identical to the original response, the request is being redirected and a different endpoint needs to be used to test.

Next add a possible delimiter between the original path and the arbitrary string. If the response is identical to the base response, the delimiter chosen was correct. If the server interperets the path with the delimiter included as part of the URL then it is incorrect.

After identifying the delimiter, add a static extension to the end of the path.

It is best to use a list of possible delimiters and list of possible extensions.

In the list of possible delimiters make sure to include encoded and decoded delimiter characters. Also include encoded non-printable characters.
