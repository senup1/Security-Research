Finding GraphQL endpoints


GraphQL APIs use the same endpoint for all requests.

If you send query{__typename} to any GraphQL endpoint, it will include the string {"data": {"__typename": “query”}} somewhere in its response. This is known as a universal query.

It is a useful tool to probe for GraphQL services.

This works because every GraphQL endpoint has a reserved field called __typename that returns the queried object's type as a string.

Here are some common GraphQL endpoints that aren't necessarily always used:

/graphql

/api

/api/graphql

/graphql/api

/graphql/graphql

The next step to find GraphQL endpoints is to test different request methods.

Typically GraphQL endpoints only accept POST requests that have a content type of application/json, however, some endpoints may accept alternative methods or content types like GET or application/x-www-form-urlencoded respectively.

Once an endpoint is discovered, send test requests to understand more of its structure.
