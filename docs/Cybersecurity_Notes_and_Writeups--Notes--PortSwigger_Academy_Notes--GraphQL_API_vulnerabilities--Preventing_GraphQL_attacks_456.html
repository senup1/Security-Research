Preventing GraphQL attacks


  • If API is not intended for public use, disable introspection on it
  • Review the API's schema and make sure it doesn't expose unintended fields to the public
  • Disable suggestions
  • Make sure it doesn't expose any private user fields

For brute-force attacks specifically:

  • Limit query depth
  • Configure operation limits
  • Configure maxiumum byte limits
  • Consider implementing cost analysis for recieved queries. If the query's resource cost is too high it will be dropped

For CSRF specifically:

  • Only accept JSON over POST
  • API validates content provided matches type
  • Implement CSRF token mechanism
