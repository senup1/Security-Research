Host Header Poisoning


Host Header Poisoning occurs when the “Host” header in an HTTP request is modified to be a domain under the attackers control before being sent.

This is possible because it is often possible for the server to be unaware of what domain it is being hosted on. As a result, the web server relies on the “Host” header whenever it needs the domain for something like a transactional email.

Protection:

Take the domain name from server side config

Use relative URLs wherever possible
