Brute-forcing a stay-logged-in cookie


  1. Provided with blog website, credentials “wiener:peter”, target username “carlos”, password wordlist
  2. Intercepted login request using provided credentials and stay signed in checkbox selected
  3. Noticed stay-logged-in header
  4. Tried decoding from base64 using cyberchef
  5. Resulting text showed wiener:51dc30ddc473d43a6011e9ebba6ca770
  6. Used hash-identifier on 51dc30ddc473d43a6011e9ebba6ca770
  7. Discovered it as md5 hash
  8. Checked with peter hashed to md5 and it matched
  9. Used cat wordlist.txt | xargs -I {} bash -c ‘echo -n “{}” | md5sum’ > wordlisthashed.txt to convert provided password wordlist to md5 version of each password
  10. Prepended carlos: to each hash in hashed wordlist
  11. Used cat wordlisthashed.txt | xargs -I {} bash -c ‘echo -n “{}” | base64’ > wordlist.txt to convert each line to base64
  12. Forwarded request to Intruder
  13. Logged out
  14. Changed id from wiener to carlos and set positions around stay-logged-in header value
  15. Set payload to base64 encoded list and started attack
