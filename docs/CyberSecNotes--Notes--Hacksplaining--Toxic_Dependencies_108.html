Toxic Dependencies


Toxic Dependencies is when a dependency (or multiple if you're really unlucky) you are using made by someone else contains a vulnerability.

This can happen pretty often and to large projects/sites/companies too because they don't always check the code contained in dependencies they use for vulnerabilities.

Protection:

Deploy known good version of software (don't just auto update to the latest version, upgrade deliberately)

Be careful using private dependencies (use well known, reputable ones)

Scan for compromised dependencies in use

Take part in penetration testing your service
