Buffer Overflows


Buffer Overflows occur when a program tries to write too much data into a fixed length block of memory.

In older systems the strategy was to input malicious code along with spam for overflow.

Then overwrite the return address with the overflow and redirect back to the malicious code entered.

In modern systems that don't allow executing data, find overflow, leak address of pointer or stack data, and overwrite the return address to exisiting functions in the system like system("/bin/sh") for example.

Example of jumping to custom address (kind of):

address to replace: 0x7fabc500

it may be stored like this: 0x00 0xc5 0xab 0x7f

fill buffer like this: AAAAAAAAAAAAAAAABBBB

A is to fill buffer B is placeholder for custom address jump

replace B's like this: AAAAAAAAAAAAAAAA\x00\xc5\xab\x7f

this will take you to the new address (which is the same as original in this example) of: 0x7fabc500

Prevention:

Keep everythin up to date.

Look for news on security issues with application versions being used.

Use managed services like AWS, Lambda, or Vercel to push the burden onto hosting providers.
