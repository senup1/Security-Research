SQL injection in different contexts


SQL injection attacks may be processed in any controllable input.

For examples some websites take input in JSON or XML format.

These different formats may provide different ways to obfuscate attacks that are otherwise blocked.

Example in XML:

<stockCheck>

<productId>123</productId>

<storeId>999 &#x53;ELECT * FROM information_schema.tables</storeId>

</stockCheck>
