Insufficient blacklisting of dangerous file types


File extensions like .php5, .shtml, etc. may not be blacklisted if the list is not thorough.

Devs may also try preventing execution by setting their config to prevent it.

For example, on an Apache server there is config by default at /etc/apache2/apache2.conf

If this is modified to contain: LoadModule php_module /usr/lib/apache2/modules/libphp.so

AddType application/x-httpd-php .php

Then it would allow execution.

Apache servers also sometimes contain .htaccess config files for directory specific config.

Changing the .htaccess or apache2.conf to accept php files under a different MIME type may be able to bypass restrictions on weaker websites.

Discovering what web server is being used, what config files are default on the server, and modifying/making use of them should be the priority.

Obfuscating the file extension is another method to circumvent protections.

Some methods:

.pHp (capitalization)

.php.jpg (extra extensions)

.php. (trailing characters)

%2Ephp (encoded .)

.php;.jpg (adding semicolon before second extension)

.php%00.jpg (adding null byte before second extension)

.p.phphp (target non recursive stripping)
