Error-based SQL injection


Error-based SQLi refers to cases where one is able to use error messages to either extract/infer data from the db even in blind contexts.

The method to accomplish this would be causing a conditional error that only displays if its either true or false.

Example: ‘ AND (SELECT CASE WHEN (1=2) THEN 1/0 ELSE ’a' END)='a

There may be times when error messages that are displayed are verbose and leak more information that may be helpful to an attacker.

One way to acheive this is to use the CAST() function.

Example: ' AND CAST((SELECT blank_column FROM blank_table) AS int)
