Server-Side Request Forgery


Server-Side Request Forgery (SSRF) occurs when an attacker tricks a server into making HTTP requests on their behalf to a place the attacker chooses.

Example:

If a user on discord pasted a link to a website, discord typically shows a preview of the website.

The same thing happens in many other social media platforms and websites.

Let's say instead the user enters a url like http://localhost/admin or http://1.1.1.1/latest/meta-data, then the server may print sensitive information.

It could also be used to send a bunch of requests from the server to the target url and act as a dos attack in the attackers place.

They could also probe random ip addresses and ports (most likely one at a time) to see if they are active and host anything useful to them

This can occur any time a server has to make a request to a third party server.

Some other examples:

Singe sign on (sso)

third party api call

image upload function that accepts urls

They can also just intercept and edit the HTTP requests from any of the methods mentioned above or more.

Protection:

Disable external validation URLs

Only make outgoing HTTP calls on behalf of real users (logged in users to log their activities)

Validate the URLs you do access

Keep a blocklist of domians you will never access in server side requests
