Network Sharing


The secure copy command (scp):

This can be used to transfer files over ssh

Copy a file from a local host to a remote host:

scp <filename>.<fileextension> <username>@<ip/domain>:/<remotedirectory>

Copy a file from a remote host to local host

scp <username>@<ip/domain>:/<remotefile> /<localdirectory>

to copy directories instead of files add the -r flag after scp

For remote synchronization use (rsync):

This is useful when there is an unstable network connection. It is able to remember where it left off and only send over what hasn't been sent yet rather than restarting the whole transfer.

Some flags:

-v: verbose output

-r: recursive (for directories)

-h: human readable output

-z: compresses file data during transfer (good for slow connections)

-a: preserves permissions, ownership, timestampts, etc

Examples:

Create backup directory on the same machine:

rsync -avh <localdirectoryone> <localdirectorytwo>

Sync files from a local host to a remote host:

rsync -avh /<localdirectory> <username>@<ip/domain>:/<remotedirectory>

Sync files from a remote host to local host:

rsync -avh <username>@<ip/domain>:/<remotedirectory> /<localdirectory>
