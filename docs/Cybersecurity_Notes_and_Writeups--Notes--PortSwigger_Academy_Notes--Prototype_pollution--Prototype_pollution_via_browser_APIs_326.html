Prototype pollution via browser APIs


Prototype pollution may also be possible through browser APIs. An example of this is the Fetch API.

The fetch() method accepts two arguments: URL to send request to, and an object that lets you control parts of the request such as the method, headers, body, parameters, etc.

Example of how one might send a POST request using fetch():

fetch('https://website.com/account/change-email', {

method: ‘POST’,

body: ‘user=carlos&email=carlos%40gmail.com’

})

Since only the method and body properties were explicitly defined, an attacker may be able to pollute Object.prototype with their own headers property.

If the headers were added after the instantiation of the object, then they would still be exploitable.

For example if there was a let username = data['x-username'] after the object instantiation, a payload such as ?__proto__[headers][x-username]=<img/src/onerror=alert(1)> would be able to execute.

Another way developers may try to prevent prototype pollution is through the use of the Object.defineProperty() method. This method enables you to set a non-configurable, non-writeable property directly on the affected object.

This approach is also flawed as it contains an object known as a descriptor. If this object is not set properly, an attacker may be able to bypass the defense by polluting Object.prototype with a malicious value property that would be passed to Object.defineProperty().

A good method for prototype pollution may be to google what properties an object can have.

For example, when doing this with Object.defineProperty() you may discover that there is a value property that accepts any type (bool, string, etc).
