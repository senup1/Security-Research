Server-side request forgery (SSRF)


Server-side request forgery is a web vulnerability that allows an attacker to cause the server-side applicaiton to make requests to unintended locations.

In a typical SSRF attack, the attacker may cause the server to make a connection to internal services and leak sensitive data.

Typically accessing the internal network in this situation would involve supplying a URL with a hostname like 127.0.0.1 or localhost.

These attacks are typically able to occur due to requests from a local machine being implicitly trusted.

In some cases there may be other back end systems that are only locally visible and therefore unprotected. These systems would be accessible through SSRF.
