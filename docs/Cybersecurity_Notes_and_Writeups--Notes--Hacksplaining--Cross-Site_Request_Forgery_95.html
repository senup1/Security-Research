Cross-Site Request Forgery


Cross-Site Request Forgery (CSRF) occurs when a user is tricked into interacting with a page/script on a third-party site that generates a malicious request to your site.

Easier to use when actions like making posts, comments, etc use GET requests as all the information for it would be passed through the URL of the HTTP request.

Commonly pronounced as “sea-surf”

Protection:

Ensure non-GET requests can only originate from your client side code

Try only using GET requests to view resources rather than modify them.

Using Anti-Forgerty Tokens (include secret and unique token with each HTTP response)
