Cross-Site Scripting


Cross-Site Scripting (XSS) permits a malicious user to execute arbitrary chunks of javascript when other users visit the website.

DOM based XSS:

If the # at the end of a url used to mark a specific location on the page (called a URI fragment) isnt properly sanitized it can take malicious javascript input and run it.

Also remember URI fragments are not sent to the server so there would be no way of logging the XSS event that occured.

Protection:

Input Sanitization

Reflected XSS:

If your website displays any parameteres form a GET or POST request back to the user (Example: results of search bar query showing input) then it is at risk to Reflected XSS.

Protection:

Input Sanitization
