NoSQL operator injection


Examples of NoSQL operators:

$where - matches documents that satisfy a JavaScript expression

$ne - Matches all values that are not equal to a specified value

$in - Matches all of the values specified in an array

$regex - Selects documents where values match a specified regular expression

Try submitting different operators into a range of user inputs and review the responses

In JSON, insert query operators as nested objects. For example: {"username":"test"} becomes {"username":{"$ne":"invalid"}}

For URL based inputs, try inserting via URL parameters. For example: username=test becomes username[$ne]=invalid

If the above doesn't work for URL based, Try other method types, try setting the content-type to application/json, then add JSON to the request and send
