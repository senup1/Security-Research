API recon


First, identify API endpoints. These are locations where an API receives requests about a specific resource on its server.

Example:

GET /api/books HTTP 1.1

Host: example.com

The API enpoint in the HTTP request above is /api/books. This might result in a list of books from a library.

Once you have discovered the API endpoints, test for:

  • required and optional parameteres
  • types of requests the endpoint accepts
  • rate limits and authentication mechanisms
