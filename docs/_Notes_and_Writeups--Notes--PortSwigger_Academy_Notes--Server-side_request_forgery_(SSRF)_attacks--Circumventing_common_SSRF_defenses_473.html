Circumventing common SSRF defenses


Blacklist based input filters:

Some applications may block input containing hostnames like 127.0.0.1 and localhost, or sensitive URLs like /admin

In this case, try decimal notation, hex notation, or omitting the zeroed octets. For 127.0.0.1 it would look like 2130706433, 017700000001, or 127.1 respectively.

You may also be able to register your own domain name that resolves to 127.0.0.1

Another option is to try obfuscating/encoding blocked strings or case variation.

Whitelist based input filters:

Some applications may implement whitelisting without checking the full input and just filtering a certian length or stopping the check when enough of the target passes.

You may be able to trick the filtering by using @ to embed credentials, # to indicate a URL fragment, or using a subdomain on a domain you control with the name of the expected host.

URL encoding characters may also be necessary.

Open redirection vulnerabilities may also be used to bypass filters.
