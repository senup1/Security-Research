The Filesystem


Essential System Directories:

/bin contains essential command line programs available to all users such as ls, mv

/sbin holds essential system binaries which are primarily intended for system administration

/etc is the core system configuration directory. It contains config files for the operating system and installed applications (should not contain any executables)

/lib contains essential shared library files that system binaries in /bin and /sbin depend on

/boot stores the files required for the system's boot processes

User and Application Data:

/home contains personal directories, files, and documents for each user.

/root is the home directory for the root user

/opt is reserved for optional third party application software packages (where you should install third party applications)

/usr contains user installed software and utilities (this is where system managed software ends up while opt is for manually installed software)

Dynamic and Temporary Data:

/var stores files that are expected to change in size and content, such as logs, cahces, etc

/tmp is for temp files, often cleared on system reboots

/run contains process IDs and other runtime data

Device and Mount Points:

/dev contains special device files that represent hardware components like hard drives and input devices

/media is a standard mount point for removable media like USB drives

/mnt is a generic mount point for temporarily mounting filesystems

System Information:

/proc is a virtual filesystem that provies real time information about currently running processes

/srv contains files for a web browser

To mount to a file system:

Use sudo mount -t (for type) <device type> <device location> /mnt (or another custom mount folder)

To unmount from a file system:

Use sudo umount <device location> or sudo umount /mnt (whatever folder you chose as the mounting spot)
