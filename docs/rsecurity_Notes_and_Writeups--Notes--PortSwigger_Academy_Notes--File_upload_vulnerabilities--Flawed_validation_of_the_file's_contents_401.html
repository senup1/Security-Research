Flawed validation of the file's contents


Some servers may try to vaidate the contents of a file match the file type.

In the case of images, they may try checking for dimesions or other image only attributes.

Similarly, certain file types may always contain the same specific byte sequence in their header or footer. For example, JPEG files alway sbegin with the bytes FF D8 FF.

If PUT requests are enabled on a server, send OPTIONS requests to all available enpoints to discover if vulnerable to file uploads through these requests.
