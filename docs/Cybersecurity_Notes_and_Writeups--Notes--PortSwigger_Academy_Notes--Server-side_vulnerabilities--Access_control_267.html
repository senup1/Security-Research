Access control


Access control is limiting who is authorized to perform actions or request resources.

If a user can gain access to functionality they are not permitted to access, this is a vertical privelage escalation.

In some cases, sensitive functionality is concealed behind security by obscurity as a weak method of protection. However, this is still dangerous because if for some reason the url became known to a user, it would still be accessible.

Some applications determine a users level of acces at login and store this information in a user controllable location.

Examples: hidden field, cookie, preset query parameter

Horizontal privelage escalation occurs if a user is able to access resources belonging to another user.

It is often possible that horizontal escalation to another user can result in vertical privelage escalation.
