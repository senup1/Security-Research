Authentication


Authentication vulnerabilities can allow attackers to gain access to sensitive data/functionality.

A brute-force attack is one way to exploit authorization protocols.

These attacks are typically automated using wordlists of usernames and passwords.

Brute forcing usernames is typically done using educated guesses based on predicatble patterns in emails or disclosure of usernames on any publicly available access points.

Brute forcing passwords is typically done using educated guesses based on information on users and/or predictable behavior based on the high entropy requirements of passwords.

Username enumeration can be done through the observation of login or registration forms that disclose existing account's usernames/emails.

In certain situations two-factor authentication can be bypassed entirely.

If the prompt for the 2fa code is on a seperate page then it is possible the original page is in a logged in state before they have entered the verification code. In this case it would be possible to access “logged in only” pages by directly navigating to them and not entering the 2fa code on the other page.
