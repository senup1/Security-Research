Permissions


SUIDs allow users that do not own a file to temporarily run as the owner of the file

Example:

root owns /etc/shadow

any user can technically edit shadow but only when using the passwd command to change their own password

s can be added with +s in chmod or by prepending the 3 digit number (Ex: 777) with a fourth digit of 4 for s

SGIDs are similar for groups. Still +s but prepend with 2 instead of 4 for numeric chmod
