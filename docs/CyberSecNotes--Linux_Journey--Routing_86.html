Routing


Packet Path to an External Network:

The sending host recognizes that the target IP address is not local.

The host consults its routing table, since there's no specific route for the eternal IP, it uses the default route, which points to the default gateway (a router).

For the packet info it uses its IP, destination IP, its MAC address, and the router's MAC as the destination MAC since the actual destination MAC is unknown.

The router rewrites the original MAC to be its own and sets the destination MAC to the MAC of the next hop.

This process is repeated until it arrives at the router connected to the destination's local network

That router then uses arp to deliver the packet to the right host.
