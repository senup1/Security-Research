Common defences against CSRF


  1. CSRF tokens - A unique, secret, unpredictable value that is generated by the server-side appication and shared with the client. When performing a sensitive action, teh client must includ ethe correct CSRF token in the request
  2. SameSite cookies - A mechanism that determines when a website's cookies are included in requests originating from other websites
  3. Referer-based validation - Some applications make use of the HTTP referrer header to attempt to verify that the request originated from the application's own domain. This is less affective than the CSRF token method.
