Finding hidden attack surface for SSRF vulnerabilities


Many server-side request forgery vulnerabilities can be discovered due to normal traffic using full URLs in parameters vs the use of partial URLs similar to endpoints in HTTP requests by server-side requests.

Some applications use server-side analytics software to track users. The analytics software typically visits third-party URLs that appear in the Referer header to analyze the contents of the referring sites. As a result, the Referer header is often a useful attack surface for SSRF vulnerabilities.
