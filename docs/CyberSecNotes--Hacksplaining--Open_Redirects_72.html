Open Redirects


Open Redirect vulnerabiliteis occur when the next parameter of a redirect URL is able to be modified and not checked

Example:

If you have a website where you need to login to access content and someone tries to access the content that requires login without being logged in, it can be common to queue the redirect after a successful login to the content they were trying to access. If that queued URL is modified and it is not checked, then it can lead to malicious actions by attackers.

Protection:

Disallow offsite redirects

Make sure all redirects are relative paths

Implementing interstitial pages (pages that state you are leaving the current site)
