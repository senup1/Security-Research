Determining the number of columns required


One method of determining the number of columns being returned is injection a series of ORDER BY clauses and incrementing the column index until an error occurs.

Example:

' ORDER BY 1 --

' ORDER BY 2 --

etc.

The second method involves sending a series of UNION SELECT payloads specifying a different number of null values.

Example:

' UNION SELECT NULL--

' UNION SELECT NULL, NULL --

' UNION SELECT NULL, NULL, NULL --

etc.

With this method, there will be an error until the number of NULLs matches the number of columns.

Different databases may have different syntaxes.

Oracle will require every SELECT query to have a FROM keyword and specify a valid table.

There is a built in table on Oracle called dual which can be used for this purpose.

On MySQL there needs to be a space following the double dash sequence.

etc.
