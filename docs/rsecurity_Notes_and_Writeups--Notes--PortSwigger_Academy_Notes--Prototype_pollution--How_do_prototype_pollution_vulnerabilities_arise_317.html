How do prototype pollution vulnerabilities arise?


Prototype pollution vulnerabilities typically arise when a JavaScript function recursively merges an object containing user-controlled properties into an existing object without sanitizing keys.

Successful exploitation requires the following components:

  • A prototype pollution source - any input that enables you to poison prototype objections
  • A sink - a JavaScript function or DOM element that enables arbitrary code execution
  • An exploitable gadget - any property that is passed into a sink without filtering or sanitization
