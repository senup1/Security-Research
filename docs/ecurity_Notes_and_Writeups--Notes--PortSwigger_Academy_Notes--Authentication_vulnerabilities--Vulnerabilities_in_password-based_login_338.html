Vulnerabilities in password-based login


Brute force attacks (Automated, educated guesses using wordlists) are a common method to circumvent password protection.

Two common methods to prevent brute forcing:

  • Locking account after too many failed attempts
  • Blocking user's IP address after too many login attempts in quick succession

Both methods offer some protection but flawed implementation can still lead to exploits.

For example, some an implementation of either of the above may reset the failed counter with a successful login. So, an attacker would just have to sign into their own valid account (if they have one) every so often to reset the counter.
